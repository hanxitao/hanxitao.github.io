Build github pages

hanxitao Apr 2, 2021 2021-04-02T16:40:00+08:00
Apr 15, 2021 2021-04-15T15:30:06+08:00 1 min

一、安装环境

1）ubuntu下

安装ruby

1

  sudo apt-get install ruby-full build-essential zlib1g-dev

安装jekyll和bundler

1

  gem install jekyll bundler

2）window10

下载jekyll并安装

切换镜像源

1

  gem sources --add https://gems.ruby-china.com/ --remove https://rubygems.org/

安装bundler和jekyll

1

  gem install bundler jekyll

安装webrick

1

  bundle add webrick

二、本地配置

1）在Github上fork

  1. 在jekyll主题网站jekyllthemes寻找一款适合你的主题，我选的一款主题是jekyll-theme-chirpy，接下来讲一下这个主题的配置
  2. 进入到该主题的Homepage，进入到该主题的github页面，然后点击fork

2）初始化

  • 克隆到本地
    1

      git@github.com:hanxitao/hanxitao.github.io.git
  • 安装依赖
    1

      bundle
  • 接着执行文件初始化。该脚本会删除一些文件（.travis.yml、_posts 下的文件和docs 目录）、把.github/workflows/pages-deploy.yml.hook的后缀.hook去掉，然后删除.github里的其他目录及文件、自动提交一个 Commit 以保存上述文件的更改。
    1

      bash tools/init.sh
  • 然后push到远程分支，仓库中会自动生成一个gh-pages分支

三、本地运行

  1. 安装依赖
    1

    bundle
  2. 执行
    1

    bundle exec jekyll s
  3. 访问
    1

    http://127.0.0.1:4000/

四、部署到github pages

  1. 修改仓库的名字为github_username.github.io
  2. 回到github上的仓库，通过Settings → Options → GitHub Pages选择gh-pages分支作为发布源
  3. 通过https://github_username.github.io即可访问已搭建完成的博客
